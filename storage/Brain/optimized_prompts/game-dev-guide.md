# 用AI开发游戏 - 怎么做得更好

## 一、先定范围，再动手

AI最怕"帮我做个游戏"这种需求。必须先回答：

1. **什么类型？** 2D横版/俯视角/3D第一人称/回合制RPG/卡牌...
2. **什么引擎？** Godot/Unity/Unreal/Pygame/纯Web
3. **多大规模？** 一个关卡demo/完整小游戏/大型项目的某个模块
4. **核心玩法？** 用一句话描述：玩家做什么→得到什么反馈

**好的开头：**
```
用Godot 4做一个2D俯视角射击游戏demo。
核心玩法：玩家控制角色在房间里移动射击，消灭波次敌人。
先做：玩家移动+射击+一种敌人AI+一个房间场景。
不做：UI/音效/存档/多关卡（后面再加）。
```

## 二、拆成可运行的小块

游戏开发最忌一次写完所有代码。正确顺序：

### 第一步：能跑的空场景（5分钟）
```
创建Godot项目，一个空场景+一个可移动的方块，WASD控制。
```
→ 验证：能跑，方块能动 ✓

### 第二步：加核心机制（每次一个）
```
给玩家加射击功能：鼠标左键发射子弹，子弹直线飞行，碰到边界消失。
```
→ 验证：能射击 ✓

```
加一个敌人：红色方块，朝玩家移动，碰到玩家扣血。
```
→ 验证：敌人能追玩家 ✓

```
子弹碰到敌人，敌人消失，加10分。
```
→ 验证：能打死敌人 ✓

### 第三步：打磨
```
加波次系统：每波5个敌人，打完等3秒出下一波，每波敌人速度+10%。
```

### 第四步：美化
```
把方块替换成精灵图，加粒子效果，加UI显示血量和分数。
```

## 三、跟AI说游戏需求的模板

### 新功能：
```
游戏引擎：[Godot 4 / Unity / ...]
当前状态：[已有什么功能]
要加的功能：[具体描述]
技术约束：[用GDScript/C#/不要第三方插件/...]
验收标准：[怎么算做完了]
```

### 修bug：
```
引擎：Godot 4
问题：玩家在墙角会卡住穿模
复现步骤：1.走到右下角墙壁 2.同时按右+下 3.角色穿过墙壁
期望：被墙壁挡住
相关代码在：[文件路径]
```

### 优化性能：
```
引擎：Godot 4
问题：场景里有200个敌人时帧率从60掉到20
当前实现：每个敌人每帧用 get_node 查找玩家位置
需要：优化到200敌人保持60fps
```

## 四、AI擅长和不擅长的

### AI擅长（多用）：
- **生成样板代码**：角色控制器、敌人AI状态机、UI系统框架
- **写Shader**：描述效果，AI写GLSL/ShaderLab
- **数据结构设计**：物品系统、技能树、存档格式
- **算法实现**：寻路A*、碰撞检测、程序化生成
- **配置文件**：关卡数据JSON、敌人属性表、对话树
- **调试**：给AI报错信息，它能快速定位

### AI不擅长（少依赖）：
- **游戏设计**：好不好玩AI判断不了，需要你自己玩测
- **美术风格**：AI能写代码但不能替你决定画风
- **手感调参**：移动速度、跳跃高度、攻击判定这些要自己反复试
- **大型架构**：超过2000行的系统，AI容易丢失上下文

## 五、项目结构建议

让AI按这个结构生成代码，后续维护方便：

```
MyGame/
├── scenes/          # 场景文件
│   ├── main.tscn
│   ├── player.tscn
│   └── enemy.tscn
├── scripts/         # 代码
│   ├── player/
│   │   ├── player_controller.gd
│   │   └── player_stats.gd
│   ├── enemy/
│   │   ├── enemy_base.gd
│   │   └── enemy_ai.gd
│   ├── systems/
│   │   ├── wave_manager.gd
│   │   ├── score_manager.gd
│   │   └── save_manager.gd
│   └── ui/
│       ├── hud.gd
│       └── menu.gd
├── assets/          # 资源
│   ├── sprites/
│   ├── audio/
│   └── fonts/
└── data/            # 配置数据
    ├── enemies.json
    ├── waves.json
    └── items.json
```

跟AI说：
```
按上面的项目结构，在scripts/enemy/下创建enemy_ai.gd，
实现状态机：idle→patrol→chase→attack→dead。
chase状态用NavigationAgent2D寻路，attack状态在攻击范围内每秒攻击一次。
```

## 六、版本管理

每完成一个功能就让AI帮你：
```
当前完成了[功能名]，帮我：
1. 列出这次改了哪些文件
2. 写一句git commit message
3. 有没有需要注意的已知问题
```

这样即使后面改坏了，也能回滚。
