# 软件测试规则

## 总则
软件测试必须验证实际功能可用性，不得仅凭进程启动或窗口出现判定通过。
每个测试阶段必须截图留证，失败时截图 + 日志一并记录到 BUGS.md。

---

## 阶段一：安装/部署

### 1.1 安装方式识别

| 安装方式 | 识别特征 | 处理策略 |
|----------|----------|----------|
| 安装包 | .exe/.msi/.pkg | 静默安装或自动点击安装向导 |
| 解压即用 | .zip/.7z/.tar.gz | 解压到目标目录 |
| 包管理器 | pip/npm/cargo/scoop | 执行安装命令 |
| 平台安装 | Steam/Epic/WeGame | 调用平台协议 |
| 源码编译 | CMake/Make/Cargo | 执行构建流程 |

### 1.2 安装流程

```
1. 检查安装前置条件（磁盘空间、依赖、权限）
2. 执行安装
3. 等待安装完成（超时 300s）
4. 验证安装结果：
   a. 可执行文件存在
   b. 注册表/快捷方式已创建
   c. 版本号正确
5. 截图记录安装完成
```

---

## 阶段二：启动

### 2.1 启动流程

```
1. 记录启动前状态
2. 执行启动命令
3. 等待进程出现（超时 30s）
4. 等待窗口出现（超时 60s）
5. 截图确认窗口已出现
6. 记录启动耗时
```

### 2.2 首次启动处理

```
1. 可能有首次配置向导：
   - 语言选择
   - 许可协议
   - 工作目录设置
   - 插件/扩展推荐
2. 自动选择默认选项或最小配置
3. 截图记录首次启动状态
```

---

## 阶段三：核心功能测试

### 3.1 文件操作类软件（IDE/编辑器）

```
1. 创建新文件/项目
2. 编辑内容
3. 保存文件
4. 打开已有文件
5. 搜索/替换功能
6. 关闭文件/项目
7. 每步截图验证
```

### 3.2 开发工具类软件

```
1. 创建/导入项目
2. 构建项目
3. 运行/调试
4. 查看输出/日志
5. 使用核心工具功能
6. 每步截图验证
```

### 3.3 图形/设计类软件

```
1. 创建新画布/场景
2. 使用基本工具（画笔/选择/移动）
3. 导入素材
4. 应用效果/滤镜
5. 导出/保存
6. 每步截图验证
```

### 3.4 通信/网络类软件

```
1. 登录/连接
2. 发送消息
3. 接收消息
4. 文件传输
5. 设置修改
6. 每步截图验证
```

### 3.5 系统工具类软件

```
1. 扫描/检测
2. 执行操作
3. 查看结果
4. 配置修改
5. 每步截图验证
```

---

## 阶段四：插件/扩展测试

```
1. 打开插件管理界面
2. 安装测试插件
3. 验证插件已加载
4. 使用插件功能
5. 卸载插件
6. 验证插件已移除
7. 每步截图验证
```

---

## 阶段五：稳定性验证

```
1. 持续运行 3 分钟
2. 每 30s 检查内存占用
3. 执行 5 次重复操作验证无内存泄漏
4. 检查日志错误
5. 截图确认运行正常
```

---

## 阶段六：退出与清理

```
1. 保存当前状态
2. 正常退出
3. 等待进程结束（超时 15s）
4. 如果未结束，强制终止
5. 收集日志
6. 生成测试报告
```
