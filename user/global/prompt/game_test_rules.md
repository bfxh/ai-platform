# 游戏测试规则

## 总则
游戏测试必须以玩家视角执行，不得仅凭文件存在或名称匹配判定通过。
每个测试阶段必须截图留证，失败时截图 + 日志一并记录到 BUGS.md。

---

## 阶段一：启动

### 1.1 启动方式识别

| 启动方式 | 识别特征 | 处理策略 |
|----------|----------|----------|
| 直接启动 | 双击 exe 即进入游戏 | 直接启动进程 |
| 启动器启动 | 先弹出启动器窗口，需点击"启动"按钮 | 识别启动器窗口 → 点击启动 → 等待游戏窗口 |
| 平台启动 | Steam/Epic/WeGame 等平台 | 调用平台协议链接或平台 CLI |
| 脚本启动 | .bat/.sh/.py 包装 | 执行脚本 |

### 1.2 启动流程

```
1. 记录启动前状态（进程列表、窗口列表）
2. 执行启动命令
3. 等待进程出现（超时 30s）
4. 等待游戏窗口出现（超时 120s）
   - 每秒检查窗口列表
   - 匹配窗口标题关键词
5. 截图确认窗口已出现
6. 记录启动耗时
```

### 1.3 启动器处理

```
1. 启动启动器进程
2. 等待启动器窗口出现（超时 30s）
3. 截图分析启动器界面
4. 定位"启动/开始/播放"按钮
5. 点击启动按钮
6. 等待启动器窗口消失或游戏窗口出现
7. 如果启动器有更新提示：
   - 记录更新信息
   - 等待更新完成（超时 300s）
   - 更新完成后重新点击启动
```

### 1.4 启动失败判定

| 情况 | 判定 | 处理 |
|------|------|------|
| 进程未出现 | 启动失败 | 记录错误，截图 |
| 窗口未出现 | 加载失败 | 检查进程是否崩溃 |
| 弹出错误对话框 | 启动错误 | 截图对话框，记录错误文本 |
| 启动器卡住 | 启动器故障 | 截图，尝试关闭重启 |

## 阶段二：主菜单导航

### 2.1 等待主菜单加载

```
1. 等待游戏窗口内容稳定（连续 3 秒截图无变化）
2. 截图确认主菜单已显示
3. 识别主菜单元素（按钮、文字、背景）
4. 如果有加载进度条，等待进度条完成
```

### 2.2 主菜单操作

```
1. 截图分析主菜单布局
2. 识别可用选项（单人/多人/设置/退出等）
3. 根据测试目标选择对应选项
4. 点击后等待响应（超时 10s）
5. 截图确认界面已切换
```

### 2.3 常见主菜单类型

| 类型 | 示例 | 操作 |
|------|------|------|
| 标准菜单 | Minecraft | 点击"单人游戏" |
| 角色选择 | MMO 类 | 创建/选择角色 |
| 存档选择 | 开放世界 | 选择已有存档或创建新世界 |
| 模式选择 | 多模式游戏 | 选择测试模式 |
| 剧情菜单 | 剧情游戏 | 点击"新游戏"或"继续" |

---

## 阶段三：进入游戏场景

### 3.1 创建/加载世界

```
1. 在主菜单点击进入游戏选项
2. 如果需要创建世界：
   a. 输入世界名称
   b. 选择游戏模式
   c. 确认创建
3. 如果需要选择存档：
   a. 定位目标存档
   b. 点击加载
4. 等待世界加载（超时 120s）
   - 检查加载进度条
   - 检查日志中的加载完成标志
5. 截图确认已进入游戏场景
```

### 3.2 加载完成判定

| 判定方式 | 适用场景 | 实现 |
|----------|----------|------|
| 日志关键字 | Minecraft 等 | "Saving chunks" / "Preparing spawn" |
| HUD 出现 | 大多数游戏 | 截图识别血条/物品栏/小地图 |
| 画面稳定 | 通用 | 连续截图对比，变化率低于阈值 |
| 进程内存 | 高级 | 读取游戏内存确认场景加载 |
| 窗口焦点 | 通用 | 确认游戏窗口获得焦点 |

## 阶段四：基础操控测试

### 4.1 移动测试

```
1. 截图记录初始位置
2. 执行移动操作：
   a. 前进 (W/↑) 2秒
   b. 停止
   c. 截图对比位置变化
   d. 后退 (S/↓) 2秒
   e. 停止
   f. 截图对比位置变化
   g. 左移 (A/←) 2秒
   h. 停止
   i. 截图对比位置变化
   j. 右移 (D/→) 2秒
   k. 停止
   l. 截图对比位置变化
3. 验证：至少 3 个方向的截图有明显位移
4. 记录移动是否流畅（帧率感知）
```

### 4.2 视角测试

```
1. 鼠标向右移动 → 视角右转
2. 鼠标向左移动 → 视角左转
3. 鼠标向上移动 → 视角上抬
4. 鼠标向下移动 → 视角下压
5. 截图对比视角变化
```

### 4.3 跳跃测试

```
1. 按跳跃键 (Space)
2. 等待落地
3. 截图确认角色位置变化
4. 重复 3 次确认稳定性
```

### 4.4 交互测试

```
1. 靠近可交互物体
2. 按交互键 (E/F)
3. 确认交互响应（UI 弹出/物品拾取/门打开等）
4. 截图记录交互结果
```

## 阶段五：玩法测试

### 5.1 战斗系统

```
1. 寻找敌对目标
2. 执行攻击操作（左键/右键）
3. 验证伤害数字/血条变化
4. 测试防御/闪避
5. 测试技能释放
6. 截图记录战斗过程
```

### 5.2 物品系统

```
1. 打开背包/物品栏
2. 检查物品显示是否正常
3. 尝试装备/使用物品
4. 尝试丢弃物品
5. 尝试合成/制作
6. 截图记录物品操作
```

### 5.3 建造/放置系统

```
1. 选择建造/放置模式
2. 放置方块/建筑
3. 验证放置是否成功
4. 尝试破坏已放置的物体
5. 截图记录建造结果
```

### 5.4 任务/对话系统

```
1. 接受任务/触发对话
2. 阅读对话内容
3. 做出选择
4. 验证任务追踪更新
5. 截图记录任务状态
```

### 5.5 模组/插件功能测试

```
1. 确认模组已加载（模组列表/日志）
2. 测试模组核心功能
3. 测试模组物品是否可获取/使用
4. 测试模组生物是否正常生成
5. 测试模组与世界/其他模组的交互
6. 截图记录模组功能
```

## 阶段六：稳定性验证

### 6.1 运行时检查

| 检查项 | 方法 | 频率 |
|--------|------|------|
| 帧率 | 截图时间差推算 | 每 10s |
| 内存占用 | 读取进程内存 | 每 30s |
| 崩溃检测 | 进程是否存活 | 每 5s |
| 日志错误 | 扫描日志 FATAL/ERROR | 每 15s |
| 窗口状态 | 窗口是否无响应 | 每 10s |

### 6.2 长时间运行

```
1. 保持游戏运行 5 分钟
2. 每分钟截图一次
3. 持续监控日志错误
4. 结束时截图确认游戏仍正常
5. 记录总运行时间和错误数
```

---

## 阶段七：退出与清理

```
1. 保存游戏（如有自动保存则等待）
2. 退出到主菜单
3. 退出游戏
4. 等待进程结束（超时 30s）
5. 如果进程未结束，强制终止
6. 收集日志文件
7. 生成测试报告
```
