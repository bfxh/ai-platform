# 插件与模组测试规则

## 总则
插件/模组测试必须验证实际功能生效，不得仅凭配置文件存在或列表中出现名称判定通过。
必须以玩家/用户视角验证：物品能否使用、生物能否生成、功能是否触发。

---

## 阶段一：加载验证

### 1.1 加载检测

| 检测方式 | 适用场景 | 实现 |
|----------|----------|------|
| 模组列表 | Minecraft Forge/Fabric | 游戏内模组菜单截图 |
| 启动日志 | 通用 | 扫描日志中模组名 + "loaded" |
| 插件管理器 | IDE/软件 | 插件列表中显示已启用 |
| 配置文件 | 通用 | 检查配置中模组/插件状态 |

### 1.2 加载流程

```
1. 确认模组/插件文件存在于正确目录
2. 启动宿主程序
3. 等待加载完成
4. 检查加载日志：
   a. 无 FATAL/ERROR 级别错误
   b. 模组/插件名出现在 loaded 列表
   c. 无依赖缺失警告
5. 截图确认加载状态
6. 如果加载失败：
   a. 记录错误信息
   b. 截图错误提示
   c. 写入 BUGS.md
```

### 1.3 依赖检查

```
1. 读取模组/插件的依赖声明
2. 验证每个依赖是否已安装
3. 验证依赖版本是否兼容
4. 如果有缺失依赖：
   a. 自动下载安装缺失依赖
   b. 重新加载验证
5. 记录依赖关系图
```

## 阶段二：功能验证

### 2.1 游戏模组功能验证

#### 物品类模组

```
1. 进入游戏世界
2. 打开创造模式物品栏
3. 搜索模组物品（按模组名/标签筛选）
4. 取出模组物品到背包
5. 切换到生存模式
6. 尝试使用物品：
   a. 右键使用（食物/工具/武器）
   b. 放置（方块/机器）
   c. 装备（盔甲/饰品）
7. 验证物品效果是否生效
8. 截图记录每个步骤
```

#### 生物类模组

```
1. 进入游戏世界
2. 使用创造模式蛋生成模组生物
3. 观察生物 AI 行为：
   a. 移动是否正常
   b. 攻击/防御是否正常
   c. 掉落物是否正确
4. 在自然环境中等待生物生成
5. 验证生成条件是否符合模组描述
6. 截图记录生物和交互
```

#### 机制类模组

```
1. 进入游戏世界
2. 触发模组机制：
   a. 生命/魔法系统：受伤/恢复
   b. 技能系统：释放技能
   c. 合成系统：使用模组配方
   d. 世界生成：探索模组结构
3. 验证机制是否按预期工作
4. 截图记录机制触发和效果
```

#### 世界生成类模组

```
1. 创建新世界（确保使用模组）
2. 探索世界寻找模组生成内容：
   a. 新生物群系
   b. 新结构/建筑
   c. 新矿物/资源
3. 使用 /locate 命令快速定位（如可用）
4. 传送至目标位置
5. 截图记录生成内容
```

### 2.2 软件/IDE 插件功能验证

```
1. 打开宿主软件
2. 确认插件已加载（插件列表/状态栏）
3. 触发插件功能：
   a. 命令面板执行插件命令
   b. 右键菜单使用插件选项
   c. 快捷键触发
   d. 侧边栏/面板交互
4. 验证功能输出是否正确
5. 测试插件配置修改
6. 截图记录每个步骤
```

## 阶段三：兼容性测试

### 3.1 模组间兼容

```
1. 同时加载多个模组
2. 测试模组间交互：
   a. 模组 A 的物品能否用于模组 B 的配方
   b. 模组 A 的生物是否与模组 B 的世界生成冲突
   c. 模组间的 UI 是否重叠/遮挡
3. 检查日志是否有冲突警告
4. 截图记录兼容性测试结果
```

### 3.2 版本兼容

```
1. 记录宿主程序版本
2. 记录模组/插件声明支持的版本范围
3. 验证是否在支持范围内
4. 如果版本不匹配：
   a. 记录不匹配信息
   b. 尝试运行并记录行为
   c. 写入 BUGS.md
```

### 3.3 性能影响

```
1. 不加载模组时记录基准帧率/响应时间
2. 加载模组后记录帧率/响应时间
3. 计算性能影响百分比
4. 如果性能下降超过 30%：
   a. 记录为性能问题
   b. 写入 BUGS.md
```

---

## 阶段四：配置测试

```
1. 修改模组/插件配置
2. 重启宿主程序
3. 验证配置是否生效
4. 测试边界配置值
5. 恢复默认配置
6. 截图记录配置修改效果
```

---

## 阶段五：卸载测试

```
1. 移除模组/插件文件
2. 重启宿主程序
3. 验证：
   a. 程序正常启动（不崩溃）
   b. 相关功能已移除
   c. 无残留错误日志
   d. 存档/项目未损坏
4. 截图记录卸载后状态
```
