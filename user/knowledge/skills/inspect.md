# Inspect Skill - 巡检技能

## 功能
定期审计知识系统，发现并修复结构问题。

## 使用方法
```
请对知识系统进行一次巡检：
[可选：指定目录或主题]
```

## AI 行为规范

### 巡检流程
1. **扫描目录结构** - 检查文件组织
2. **检查概念冲突** - 发现重复/冲突定义
3. **发现孤岛文件** - 找出没有被引用的文件
4. **验证来源支持** - 检查结论是否有来源
5. **生成报告** - 保存到 inspection/

### 核心原则
- 巡检先出报告，不要默认自动改
- 巡检最好按目录或主题分批进行
- 巡检关注结构问题，不只看内容对不对
- 巡检结果也要落文件

## 检查清单

### 1. 定义冲突
- 检查 concepts/ 中是否有相似定义的概念
- 检查同一概念的不同别名是否指向不同含义

### 2. 重复概念
- 检查是否有多个文件描述同一个概念
- 检查 summaries/ 中是否有高度相似的内容

### 3. 孤岛文件
- 检查哪些文件没有被其他文件引用
- 检查 index/ 中是否遗漏了重要文件

### 4. 来源缺失
- 检查哪些结论没有 source 字段
- 检查过时的 summary 是否需要更新

### 5. 结构问题
- 检查目录是否过于深层
- 检查命名是否一致
- 检查是否有空文件

## 巡检频率建议

- **周检**：快速扫描，发现明显问题
- **月检**：深度巡检，生成完整报告
- **季度检**：系统评估，考虑重构
