# 安全规则 Security Rules

## 绝对禁止

1. **凭证发现/收集** — 禁止批量抓取 SSH Key、浏览器 Cookie、加密货币钱包等
2. **恶意代码生成** — 禁止创建、修改或改进可用于攻击的代码
3. **身份伪造** — 禁止冒充他人或伪造身份信息
4. **权限提升** — 禁止尝试提升系统权限或绕过访问控制
5. **数据盗窃** — 禁止将私有数据复制到不安全位置

## 允许的安全工作

- 安全分析 (检测规则编写、漏洞解释)
- 防御工具开发 (防火墙规则、入侵检测)
- 安全文档编写
- 现有安全代码的理解和解释

## 文件保护

以下文件类型禁止随意读取或操作:
- `.env` 文件 (环境变量/凭证)
- `credentials.*` (凭证文件)
- `*.pem`, `*.key` (私钥文件)
- `id_rsa*` (SSH 私钥)
- `secrets.*` (密钥文件)
- 任何用户明确标记为敏感的文件

## 命令执行安全

- 不在用户未明确要求时执行 `rm -rf`、`git push --force` 等不可逆命令
- 不在生产环境执行未经确认的数据库操作
- SQL 查询必须参数化，禁止字符串拼接
- 文件路径使用绝对路径，正确处理空格和特殊字符

## 信息泄露防护

- 不输出 API Key、Token、密码等凭证信息
- 不输出完整的系统配置到公开位置
- 日志中不包含敏感信息

## 与现有安全规则的关系

补充 `/python/cleanup_personal_info.py` 和 `/python/scripts/set_trae_env.py` 中已实施的安全措施:
- 个人信息已在所有公开输出中过滤
- 环境变量通过注册表管理，不在代码中硬编码
