# 代码审查 Code Review

## 执行步骤

1. **读取变更**: 获取所有修改的文件和变更内容
2. **理解上下文**: 阅读相关模块的现有代码，理解设计意图
3. **逐文件审查**:
   - 正确性: 逻辑是否有误？边界条件是否处理？
   - 安全性: 是否存在注入、XSS、权限绕过？
   - 性能: 是否有不必要的循环、重复查询、内存泄漏？
   - 可维护性: 代码是否清晰？命名是否合理？
4. **生成报告**: 列出发现的问题，按严重性排序:

| 级别 | 符号 | 含义 |
|------|------|------|
| P0 | [!!] | 阻塞级 — 必须修复才能合入 |
| P1 | [!] | 高危 — 强烈建议修复 |
| P2 | [-] | 建议 — 可以后续优化 |
| P3 | [i] | 信息 — 仅供参考 |

## 审查检查清单

- [ ] 是否存在硬编码的敏感信息？
- [ ] 用户输入是否经过验证和参数化？
- [ ] 错误处理是否恰当 (无 bare except)？
- [ ] 是否遵循 coding-style.md 规范？
- [ ] 是否有循环导入风险？
- [ ] 是否有重复代码可以抽取？
- [ ] 函数是否单一职责？
- [ ] 测试是否覆盖主要路径？
