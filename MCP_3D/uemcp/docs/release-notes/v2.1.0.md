# UEMCP v2.1.0 Release Notes

**Release Date**: 2026-03-26

## Highlights

- Full migration from deprecated `EditorLevelLibrary` to modern UE subsystem APIs
- MCP SDK updated to 1.28.0
- Per-operation configurable timeouts prevent 504 errors on long operations
- Corrected tool manifest type annotations for accurate MCP schemas

## Breaking Changes

- **Server runtime requirement**: The MCP server now requires **Node.js >= 20.19.0** (see `server/package.json`). Users on Node 18/19 must upgrade Node.js or remain on UEMCP v2.0.x.

## UE API Modernization

Migrated all `EditorLevelLibrary` static method calls to their modern subsystem equivalents, ensuring compatibility with UE 5.5+ where these APIs are deprecated:

| Deprecated API | Replacement |
|---|---|
| `EditorLevelLibrary.spawn_actor_from_class()` | `EditorActorSubsystem.spawn_actor_from_class()` |
| `EditorLevelLibrary.spawn_actor_from_object()` | `EditorActorSubsystem.spawn_actor_from_object()` |
| `EditorLevelLibrary.destroy_actor()` | `EditorActorSubsystem.destroy_actor()` |
| `EditorLevelLibrary.save_current_level()` | `LevelEditorSubsystem.save_current_level()` |
| `EditorLevelLibrary.pilot_level_actor()` | `LevelEditorSubsystem.pilot_level_actor()` |
| `EditorLevelLibrary.eject_pilot_level_actor()` | `LevelEditorSubsystem.eject_pilot_level_actor()` |
| `EditorLevelLibrary.set_level_viewport_realtime()` | `LevelEditorSubsystem.editor_set_viewport_realtime()` |

Updated across: plugin code, tests, documentation, and example scripts.

## Configurable Timeouts

Operations now have per-command timeout defaults instead of a blanket 10-second limit:

- `viewport_screenshot`: 30s
- `asset_import_assets`: 60s
- `batch_operations` / `actor_batch_spawn`: 30s
- `blueprint_compile` / `blueprint_create` / `blueprint_document`: 30s
- `python_proxy`: 30s
- Material creation: 20s
- All other commands: 10s (unchanged)

The Python listener also accepts an explicit `timeout` field in the command JSON for override.

## Tool Manifest Improvements

Added missing Python type annotations to operation methods, fixing incorrect JSON Schema types in the MCP tool manifest:

- `ue_logs.lines` now correctly reports as `number` (was `string`)
- `get_level_actors.limit` now correctly reports as `number`
- `get_outliner_structure.showEmpty` now correctly reports as `boolean`
- `look_at_target` numeric parameters now correctly report as `number`
- Various `Optional[str]` parameters now correctly nullable
