# Release v3.8.0

Released: 2026-04-02

## Features

- **Niagara VFX System** (v3.2) — 8 new MCP tools: create systems, add emitters/modules, configure renderers, compile, spawn actors, set parameters, list templates. Rewritten for UE 5.7 Python API
- **Performance Profiling** (v3.3) — 3 new MCP tools: `perf_rendering_stats`, `perf_gpu_stats`, `perf_scene_breakdown` for scene analysis and rendering cost inspection
- **Animation Blueprint System** (v3.4) — 8 new MCP tools: create ABPs, state machines, states, transitions, variables, montages, layer linking, and metadata inspection
- **Advanced Materials & Audio** (v3.5) — 10 new MCP tools: material graph editing (add/connect expressions, set properties, create functions, inspect graph), MetaSound asset creation and node routing
- **Data & Procedural Systems** (v3.6) — 12 new MCP tools: DataTable CRUD (create/add/get/update/delete rows), custom struct and enum creation, Enhanced Input System mapping
- **PCG & StateTree AI** (v3.7) — 14 new MCP tools: PCG graph creation and node management (195+ node types), StateTree asset authoring with states, transitions, tasks, evaluators, and bindings
- **Mesh & LOD Management** (v3.8) — 5 new MCP tools: mesh metadata inspection, FBX LOD import, LOD screen-size thresholds, auto LOD generation, per-LOD instance breakdown

## Testing

- 310 Python unit tests across all ops modules (up from 51 in v3.1.0)
- Phase 7 smoke tests in `test-comprehensive-mcp.js` covering all 52 new tools from v3.3–v3.8
- `testToolStrict()` helper reduces error-check boilerplate in comprehensive E2E tests

## Bug Fixes

- PCG plugin crash on `restart_listener()`: `unreal.PCGGraphInterface` type annotation evaluated at import time caused `AttributeError`. Fixed with `from __future__ import annotations` + `hasattr` import guard
- PCG tools registered conditionally — gracefully skipped when PCG plugin absent (mirrors Niagara pattern)
- `test_performance` no longer overwrites shared `unreal` mock across test modules
- Restored closing triple-quote in `ops/__init__.py` docstring
- Niagara `set_parameter` complexity reduced — extracted typed dispatch helpers to satisfy linter
- E2E log scanner no longer flags expected error-handling test failures as CI failures
